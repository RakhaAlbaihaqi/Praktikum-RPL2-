PERTEMUAN 1 : berisi program Java sederhana yang digunakan untuk melatih konsep dasar pemrograman. Program tersebut melakukan beberapa operasi dasar, seperti melakukan penjumlahan dua angka dan menampilkan hasilnya, meminta pengguna memasukkan nama kemudian menampilkan sapaan, melakukan perulangan untuk menampilkan angka dari 0 hingga 10, serta melakukan pengecekan kondisi menggunakan percabangan untuk menentukan apakah sebuah angka bernilai positif, negatif, atau nol. Seluruh bagian program menunjukkan penggunaan variabel, input-output, loop, dan struktur kontrol sebagai fondasi utama dalam pemrograman Java.
PERTEMUAN 2 : pemahaman konsep dasar Object-Oriented Programming (OOP) dalam Java, khususnya mengenai struktur dan anatomi sebuah class. Pada project tersebut dijelaskan bahwa sebuah class memiliki komponen penting seperti atribut yang berfungsi menyimpan data, serta method yang berisi perilaku atau aksi yang dapat dilakukan oleh objek. Contoh class sederhana bernama Mahasiswa ditampilkan dengan atribut nama dan umur, serta method untuk menampilkan informasi tersebut. Project ini membantu memahami bagaimana class digunakan sebagai blueprint untuk membuat objek dan bagaimana atribut maupun method bekerja di dalamnya.
PERTEMUAN 3 : aplikasi CRUD sederhana untuk mengelola data mahasiswa menggunakan Java, MySQL, dan konsep MVC. Melalui kelas MahasiswaDAO, aplikasi terhubung langsung ke database menggunakan JDBC untuk melakukan operasi tambah, baca, ubah, dan hapus data mahasiswa. MahasiswaController bertugas memproses data dan menampilkan informasi ke konsol, sementara MahasiswaView menyediakan antarmuka berbasis teks (CLI) yang memungkinkan pengguna memilih menu, memasukkan data, dan melihat hasilnya secara langsung. Aplikasi ini juga dilengkapi fitur pengecekan koneksi database dan mekanisme looping menu sehingga pengguna dapat terus melakukan operasi tanpa keluar dari program.
PERTEMUAN 4 : menunjukkan halaman identitas tanpa isi materi lanjutan. Meskipun demikian, secara umum ACT 4 dalam praktikum RPL 2 biasanya berisi penerapan konsep dasar rekayasa perangkat lunak, seperti pembuatan class dan objek, analisis kebutuhan sederhana, atau implementasi fungsi dasar terkait modul pembelajaran. Karena isi lengkapnya tidak terbaca, proyek ini dapat dipahami sebagai bagian dari latihan pemahaman konsep RPL yang mendukung materi pada pertemuan sebelumnya.
PERTEMUAN 5 : konsep Dependency Injection (DI) dalam Spring Boot, yaitu teknik di mana sebuah objek tidak dibuat secara langsung oleh kelas yang membutuhkannya, tetapi disediakan oleh Spring melalui proses injection. Penjelasan dalam project ini menekankan bahwa DI sangat penting karena dapat mengurangi tingkat ketergantungan antar kelas (low coupling), mempermudah unit testing dengan memungkinkan penggunaan mock, serta membuat manajemen objek menjadi lebih otomatis melalui Spring Container. Selain itu, DI mendukung prinsip desain SOLID, khususnya Inversion of Control, sehingga pengembangan aplikasi menjadi lebih terstruktur dan mudah dipelihara. Project ini juga menunjukkan penggunaan anotasi penting seperti @Autowired, @Service, dan @Repository sebagai bagian dari implementasi DI dalam Spring Boot.
PERTEMUAN 6 : aplikasi manajemen data mahasiswa yang menggunakan Java Spring Boot, Hibernate/JPA, dan MySQL untuk menjalankan proses CRUD seperti menambah, menampilkan, mengubah, dan menghapus data mahasiswa. Data ditampilkan melalui GUI Swing yang memungkinkan pengguna memasukkan data dan melihatnya dalam tabel. Struktur MVC digunakan untuk memisahkan model, layanan logika, controller, serta tampilan, sehingga aplikasi berjalan teratur dan mudah dikembangkan.
